# 前呈似景供应链小程序开发记录

## 当前实现状态

- 已初始化微信原生小程序项目结构。
- 已接入云开发目录和云函数骨架。
- 已实现 mock-first 数据模式：没有正式云环境资料时，页面使用本地 Storage 数据跑通流程。

## 开发运行

1. 用微信开发者工具打开本目录。
2. AppID 暂时是 `touristappid`，拿到正式 AppID 后更新 `project.config.json`。
3. 当前不依赖真实云环境即可预览流程。
4. 需要接真实云开发时，先创建云环境，再上传 `cloudfunctions/bootstrap` 和 `cloudfunctions/api`。

## 云开发集合规划

- `users`：客户/用户资料。
- `admins`：管理员手机号与角色。
- `products`：商品资料。
- `categories`：一级分类。
- `orders`：订单与订单商品快照。
- `addresses`：收货地址。
- `audit_logs`：审核记录。
- `home_contents`：首页内容配置。
- `message_settings`：消息推送设置。
- `operation_logs`：操作记录。
- 对账中心依赖 `orders.settlementStatus` 字段管理待结算 / 已结算。

## 已知待深化

- Excel/CSV 解析目前是入口和审核骨架，拿到真实模板后补解析逻辑。
- 对账中心已支持 Excel 兼容导出，但仍可继续按正式打印模板优化版式。
- 消息推送目前是设置页和云函数预留，拿到模板 ID 后接订阅消息发送。
- 真实云数据库权限规则需要在微信开发者工具中配置。
- 首页和商品图片目前使用 `assets/temp/` 的临时预览素材，正式素材到位后替换。

## 图片素材规则

- `assets/temp/`：小程序可引用的临时预览素材，来自参考截图裁剪压缩，单张目标约 30-80KB。
- `D:\codex\前呈似景供应链_参考截图_不参与打包`：原始整屏截图，仅用于需求和视觉参考，不放在小程序项目目录内。
- `project.config.json` 已配置忽略 `references` 和 `docs`，避免参考资料进入小程序包。
- 正式上线前，`assets/temp/` 应替换为客户正式素材或云存储 fileID。
