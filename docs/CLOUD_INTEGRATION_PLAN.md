# 云开发接入执行计划

## 目标

在保留当前可演示样机能力的前提下，将项目从 `mock-first` 逐步切换到微信云开发真实后端，最终完成：

1. 正式 `AppID` 与云环境接入
2. 真实用户登录与角色权限识别
3. 商品、客户、订单、审核、对账等核心业务数据上云
4. 本地 mock 数据从主数据源降级为兜底/初始化数据源
5. 为后续图片正式替换、导入导出、消息通知打下稳定基础

## 已确认环境

- 小程序 `AppID`: `wx3079736104fac8e3`
- 云开发环境 ID: `a01-d4ggnjhhqfabf9ba2`

## 开发原则

1. **先接基础设施，再切业务数据**
   不直接跳到订单或审核逻辑，先确保云环境、集合结构、云函数骨架可用。

2. **分批替换，不一次性移除 mock**
   每完成一批真实接入，就保留临时兜底，避免页面整体失效。

3. **先打通高频主链路**
   首页、商品、登录、客户、订单优先；审核中心、对账中心后置。

4. **每一阶段都可回退**
   通过 Git 提交点和数据访问封装，确保切换错误时能快速回到稳定状态。
