# 项目开发标准

## 权限标准

### 看价权限

未获得看价权限的用户可以浏览：

- 首页
- 分类/选购菜单
- 商品列表
- 商品详情
- 协议说明
- 我的页基础信息

未获得看价权限的用户不能看到或执行：

- 商品列表 `+` 加购按钮
- 商品详情底部 `选购`
- 商品详情底部 `加入选购`
- 选购清单提交订单
- 结算页提交订单
- 售罄商品的加入选购动作

实现要求：

- 无权限时优先隐藏动作入口，不使用置灰按钮。
- 即使用户通过页面路径直接进入结算页，也必须再次校验权限。
- 价格统一展示为 `***/单位`。
- 售罄商品必须明确展示售罄状态，且前台不可加入选购。

## 按钮标准

所有按钮必须满足：

- 文案水平居中。
- 文案垂直居中。
- 不依赖 `line-height` 做垂直居中。
- 使用 `display: flex; align-items: center; justify-content: center;`。
- 新增主按钮优先使用 `.btn-primary`。
- 新增次按钮优先使用 `.btn-ghost`。

禁止：

- 在按钮样式里单独写 `line-height: 按钮高度` 来做居中。
- 为了对齐给文字单独加不稳定的 `padding-top`。

## 素材标准

- 参考截图放在项目目录外的 `D:\codex\前呈似景智链_参考截图_不参与打包`，不允许业务代码直接引用。
- 小程序本地临时素材放在 `assets/temp/`。
- 正式商品图、轮播图、详情图后续优先迁移到云存储。
