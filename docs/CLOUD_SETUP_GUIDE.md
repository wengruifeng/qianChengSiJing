## 第三步：初始化集合

当前初始化函数为：

- `bootstrap`

它支持两种用途：

1. 创建云数据库集合
2. 在空集合中写入 demo 演示数据

### 建议调用方式

在微信开发者工具的云函数测试面板中执行：

```json
{
  "seedDemo": true
}
```

如果当前云环境里已经有旧的演示数据，且这些数据里包含旧的本地图片路径，建议改用后台“店铺设置”中的：

- `清理演示图片路径`

它会直接把数据库里演示数据使用的本地图片路径字段清空，不删除订单、客户等业务数据，也更不容易触发云函数超时。

### 预期结果

会创建以下集合：

- `users`
- `admins`
- `products`
- `categories`
- `orders`
- `order_items`
- `carts`
- `addresses`
- `audit_logs`
- `home_contents`
- `message_settings`
- `operation_logs`

并写入一批最小可演示数据。

### 图片相关说明

默认执行：

```json
{
  "seedDemo": true
}
```

只会写**纯数据**，不会：

- 上传图片到云存储
- 把项目里的 `/assets/temp/...` 本地图片路径写进数据库

如果你确实要生成“带项目临时图路径”的演示数据，需要显式使用：

```json
{
  "seedDemo": true,
  "seedDemoWithBundledImages": true
}
```

### 注意

- 如果某些集合已存在，`bootstrap` 不会报致命错误
- 如果某些集合里已经有数据，seed 会跳过，不会重复灌 demo 数据

## seedDemo 后异常排查口径

### 1. 首页空白，但页面不报错

通常表示：

- `seedDemo` 没执行
- 或集合已建好，但仍是空表

### 2. 页面直接报云函数错误

通常表示：

- `bootstrap` 或 `api` 没上传部署成功

### 3. 前台有内容，后台没内容

通常表示：

- 管理员身份没识别上
- 或 `admins` 演示数据没有写进去

### 4. 后台入口完全没有

通常表示：

- 当前登录手机号没有命中管理员数据
