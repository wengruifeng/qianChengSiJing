## 第三步：初始化集合

当前初始化函数为：

- `bootstrap`

它支持两种用途：

1. 创建云数据库集合
2. 在空集合中写入 demo 演示数据

### 建议调用方式

在微信开发者工具的云函数测试面板中执行：

```json
{
  "seedDemo": true
}
```

### 预期结果

会创建以下集合：

- `users`
- `admins`
- `products`
- `categories`
- `orders`
- `order_items`
- `carts`
- `addresses`
- `audit_logs`
- `home_contents`
- `message_settings`
- `operation_logs`

并写入一批最小可演示数据。

### 注意

- 如果某些集合已存在，`bootstrap` 不会报致命错误
- 如果某些集合里已经有数据，seed 会跳过，不会重复灌 demo 数据

## 第四步：联调验证顺序

建议严格按下面顺序验，不要乱跳。

### 1. 登录与权限

测试账号：

- 超级管理员：`13800000000`
- 管理员：`13900000000`
- 已通过客户：`13700000000`

验证点：

- 登录成功
- “我的”页能看到正确身份
- 管理员能看到后台入口
- 普通客户没有后台入口

### 2. 首页与商品

验证点：

- 首页分类能正常显示
- 推荐和新品区能显示
- 商品详情能打开
- 价格权限符合用户身份

### 3. 客户申请与地址

验证点：

- 未获看价权限用户能提交申请
- 后台客户管理能看到申请
- 地址新增后，提交订单时可选

### 4. 购物车与下单

验证点：

- 商品列表可加购
- 商品详情可加购
- 查看选购页可改数量
- 提交订单后跳转订单详情

### 5. 后台订单与库存

验证点：

- 后台订单列表能看到新订单
- 改订单状态后前台状态同步
- 确认订单时库存真实扣减
- 取消未确认订单时锁定库存释放

### 6. 审核中心

验证点：

- 商品编辑提交后能进入审核中心
- 超级管理员通过后商品正式生效
- 拒绝后能看到拒绝状态与原因

### 7. 对账中心

验证点：

- 能筛真实订单
- 能标记已结算
- 能按选中订单导出
